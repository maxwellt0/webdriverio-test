# Senior Test Engineer — Task Instructions

You will find a single file in this zip: `index.html`. Open it in any modern browser (or serve the folder with a static server) — that is the System Under Test (SUT).

## What to do

1. **Explore.** Open the app and run a quick exploratory test to get familiar with all features and flows.
2. **Test plan.** We want **all critical scenarios** covered. Start by writing a short test plan.
3. **Test cases.** Turn the plan into concrete test cases.
4. **Automation.** Automate the **critical flows** from your test cases.
5. **Time box.** You have **at most 3 days** to deliver everything above.
6. **Optional notes.** If you want to explain *why* you chose a particular approach, feel free to add a short document — it is optional, not required.

## Tooling

- **Framework** — pick **one**: WebdriverIO, Selenium, Playwright, or Cypress.
- **Language** — pick **one**: JavaScript, TypeScript, Python, or Java.

See the email for the accepted submission methods (public GitHub link is preferred).

Good luck.
