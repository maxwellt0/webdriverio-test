### TC-I18N-01

**Language switch translates labels and flips RTL** — P0 — plan §5.9

- **Precondition**: PRE-CLEAN.
- **Steps**:
  1. Pick 3 visible labels with known en/fa translations (e.g. **Sign in**, **Difficulty**, **New Game**). Read each in `en` mode.
  2. Switch language to `fa`.
  3. Re-read the same 3 labels.
  4. Read `<html lang>` and `<html dir>`.
- **Expected**:
  - Each label reads its Persian translation from the en/fa key map.
  - `<html lang="fa">`, `<html dir="rtl">`.
- **Notes**: Use a single source of truth for the expected strings (centralized en/fa map in the suite). Do not hardcode strings in each case.

### TC-I18N-03

**Language switch mid-game does not reset the board** — P1 — plan §5.9

- **Precondition**: PRE-USER; play at least 2 human + 2 computer moves on the board.
- **Steps**:
  1. Capture the current board state (cell `data-state` values) and status.
  2. Switch language to `fa`.
  3. Capture again.
- **Expected**:
  - Board cell states identical before/after.
  - Status pill text changed to the Persian translation (it is still the same logical status).
  - Game can continue normally after the switch.

### TC-I18N-05

**History dates use the Jalali calendar in `fa` mode** — P1 — plan §5.9

- **Precondition**: PRE-USER; ≥ 1 row in history; language set to `fa`.
- **Steps**:
  1. Read the date column text from the most recent history row.
- **Expected**:
  - Text uses Persian digits.
  - Year corresponds to the current Jalali year (Gregorian – 621, ± 1 around Nowruz).
  - Month is one of the Jalali names (Farvardin / Ordibehesht / Khordad / Tir / Mordad / Shahrivar / Mehr / Aban / Azar / Dey / Bahman / Esfand) — not an English month name or Gregorian month number.
- **Notes**: Computing the expected Jalali date in JS: `new Date().toLocaleString('fa-IR')` produces the canonical reference; the test should compare to that, not a hardcoded string.
